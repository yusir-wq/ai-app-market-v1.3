# AI应用广场 V1.3 智能体迭代执行计划

> **版本：** V1.3  
> **日期：** 2026-06-18  
> **目标：** 快速批量上线功能型智能体，低成本、高数量，抢占工具类AI流量入口  
> **对标竞品：** 录咖AI (reccloud.cn)、佐糖 (picwish.cn)

---

## 一、核心策略

| 策略 | 说明 |
|------|------|
| **降维替代** | 单点功能做到极致，替代专业软件的80%使用场景 |
| **零门槛** | 用户上传/输入→一键生成→下载，3步闭环 |
| **存量复用** | 优先复用现有MCP/模型能力，不做复杂AI训练 |
| **模板驱动** | 智能体内部用Prompt模板+参数映射，降低开发成本 |

### 品类一：视频处理（4个）

## 三、快速上线技术方案

### 3.1 架构设计：通用智能体壳 + 能力插件

```
┌─────────────────────────────────────────────────────┐
│                  智能体通用壳（Frontend）               │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐ │
│  │ 上传区   │  │ 参数区   │  │ 预览区   │  │ 下载区   │ │
│  │(统一UI) │  │(动态表单)│  │(统一展示)│  │(统一出口)│ │
│  └────┬────┘  └────┬────┘  └────┬────┘  └────┬────┘ │
│       └─────────────┴─────────────┴─────────────┘     │
│                        │                              │
│              ┌─────────┴─────────┐                   │
│              │   Agent Engine    │  ← 路由+模板+执行   │
│              │  (复用V1.2架构)   │                   │
│              └─────────┬─────────┘                   │
│                        │                              │
│         ┌──────────────┼──────────────┐              │
│         ▼              ▼              ▼              │
│    ┌─────────┐   ┌─────────┐   ┌─────────┐         │
│    │MCP工具   │   │外部API  │   │本地模型  │         │
│    │(优先)   │   │(兜底)   │   │(可选)   │         │
│    └─────────┘   └─────────┘   └─────────┘         │
└─────────────────────────────────────────────────────┘
```

### 3.2 复用V1.2已有能力

| V1.2已有能力 | V1.3如何复用 |
|-------------|-------------|
| 对话优先架构 | 每个智能体就是一个"对话模板"，输入区固定为上传/表单 |
| MCP服务中心 | 将工具型智能体注册为MCP工具，可被对话直接调用 |
| 多模型聚合 | 部分智能体支持"多模型对比"（如文案生成可选多个LLM） |
| 文件上传区 | 直接复用上传组件，扩展支持视频格式 |
| 模型卡片UI | 智能体卡片复用现有ModelCard组件 |
| 分类页布局 | 智能体分类页复用现有分类网格布局 |

### 3.3 技术选型（低成本优先）

| 能力 | 方案A（推荐） | 方案B（备选） | 成本 |
|------|-------------|-------------|------|
| 语音转文字 | Whisper API / 开源Whisper本地部署 | 讯飞/百度API | 低 |
| 文字转语音 | Edge-TTS / Coqui-TTS（开源） | 讯飞/阿里云TTS | 极低 |
| 视频去水印 | OpenCV + LaMa Inpainting（开源） | 第三方API | 低 |
| 字幕生成 | Whisper + pysrt / ffmpeg | 第三方API | 低 |
| 视频生成 | 图文模板拼接（低成本）+ 外部API（高质量） | Runway/Pika API | 可控 |
| 图片修复 | GFPGAN / Real-ESRGAN（开源） | 第三方API | 低 |
| 文案生成 | 复用V1.2 LLM对话能力 | — | 零新增 |
| 总结提炼 | 复用V1.2 LLM对话能力 | — | 零新增 |

## 六、成本估算

| 项目 | 成本级别 | 说明 |
|------|---------|------|
| 开发人力 | 1名前端 + 0.5名后端 | 3周冲刺 |
| 开源模型部署 | 低（GPU服务器） | Whisper、TTS、图像修复模型 |
| API调用费 | 可控 | 仅在开源方案不足时启用第三方 |
| 存储/CDN | 中 | 音视频文件存储与分发 |
| **总计** | **低** | 以自部署开源方案为主 |

## 七、成功指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 智能体数量 | ≥10个 | V1.3上线时 |
| 单个智能体开发周期 | ≤3天 | 基于通用壳复用 |
| 用户端到端操作步骤 | ≤3步 | 上传→处理→下载 |
| 首屏加载时间 | ≤2s | 智能体列表页 |
| 用户转化率 | ≥15% | 访问→使用至少一个智能体 |

## 八、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 视频处理性能不足 | 高 | 限制文件大小/时长，排队处理，异步通知 |
| 开源模型效果不达标 | 中 | 准备第三方API兜底方案 |
| 10个智能体3周完不成 | 中 | 按Phase拆分，可独立发布 |
| 与V1.2架构冲突 | 低 | 智能体作为MCP工具注册，解耦部署 |

## 1. 迭代概述

### 1.1 版本定位

V1.2 是一次**架构级升级**，核心目标是从「模型优先」架构迁移到「对话优先」架构，同时补齐多模型并行对话、联网搜索展示、深度思考展示、移动端适配等关键能力。

### 1.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **对话优先** | 对话是核心实体，模型是对话中可选择的资源 |
| **多模型并行** | 单轮对话支持 1~4 个模型同时回复 |
| **渐进增强** | 联网搜索、深度思考等功能嵌套在模型卡片内 |
| **移动优先响应式** | 768px 断点，左侧导航变为滑动抽屉 |

### 1.3 需求清单总览

| # | 功能模块 | 需求名称 | 优先级 |
|---|---------|---------|--------|
| 1 | 全局调整 | 结构布局调整 — 对话优先架构迁移 | P0 |
| 2 | 工作台-对话区 | 多模型聚合对话 | P0 |
| 3 | 工作台-对话区 | 联网搜索结果展示 | P1 |
| 4 | 工作台-对话区 | 深度思考结果展示 | P1 |
| 5 | 工作台-对话区 | 上传区调整 | P1 |
| 6 | 工作台-对话区 | 图片区、视频区交互调整 | P1 |
| 7 | 全局功能 | 移动端适配 | P0 |
| 8 | 全局功能 | 邀请奖励系统（病毒式传播） | P1 |

## 2. 核心架构升级

### 2.1 架构迁移：模型优先 → 对话优先

**V1.1 架构（模型优先）：**

```
用户选择模型 → 进入模型对话页 → 发送消息 → 单模型回复
```

**V1.2 架构（对话优先）：**

```
用户进入首页 → 发送消息 + @提及模型 → 多模型并行回复 → 对话历史保存
```

## 3. 功能变更详情

### 3.1 多模型聚合对话（P0）

#### 3.1.1 选模型方式

用户在输入框中输入 `@` 触发模型提及弹窗，可多选 1~4 个模型。已选模型显示为输入框上方 pill 标签。

#### 3.1.2 消息发送流程

1. 用户输入消息 + @模型（1~4 个），点击发送
2. 创建用户消息气泡（右侧，显示 @的模型名称）
3. 为每个被提及的模型创建独立的 AI 回复卡片
4. 模型回复**并行生成**（模拟场景使用 staggered setTimeout：800ms + index × 1200ms）
5. 全部完成后输入框恢复可用

#### 3.1.3 卡片网格布局

| 模型数量 | 布局列数 |
|---------|---------|
| N=1 | 1 列 |
| N=2 | 2 列 |
| N=3 | 3 列 |
| N=4 | 4 列 |

使用 CSS Grid：`grid-template-columns: repeat(N, 1fr)`

#### 3.1.4 模型回复卡片设计

```
┌──────────────────────────────────────┐
│ 🤖 DeepSeek V4 Pro    联网 ✓  深度思考 ✓ │
│ ──────────────────────────────────── │
│ [深度思考过程 - 默认折叠]              │
│                                      │
│ 回复正文内容…                          │
│                                      │
│ 引用来源 (3)                          │
│                                      │
│ ⏱ 2.4s    [复制] [重新生成] [回复]     │
└──────────────────────────────────────┘
```

**卡片操作：**

- **回复按钮**：点击后，后续对话只针对此模型（其他模型不再参与）
- **复制 + 重新生成**：仅作用于当前卡片
- **联网/深度思考徽章**：根据本轮发送时的开关状态显示
- **加载态**：显示"思考中…"动画 + 骨架屏

### 3.2 联网搜索结果展示（P1）

#### 3.2.1 UI 表现

- AI 回复卡片底部显示 `引用来源 (N)` 文字按钮（N 为结果总数）
- 点击后**从右侧抽屉**弹出引用来源列表
- 抽屉内容：每条结果包含网站名称、标题、简介、可点击的 URL 链接

#### 3.2.2 抽屉规格

| 属性 | 规格 |
|------|------|
| 宽度 | 380px |
| 标题 | "引用来源" |
| 内容 | 列表滚动，每条结果带外链图标（↗） |

#### 3.2.3 数据结构

```typescript
interface SearchResult {
  id: string;
  siteName: string;
  title: string;
  snippet: string;
  url: string;
}
```

### 3.3 深度思考结果展示（P1）

#### 3.3.1 UI 表现

- AI 回复卡片正文前增加「深度思考过程」区块
- **默认折叠**，只显示可展开的标题行
- 点击展开后显示模型推理内容（等宽字体、浅色背景）

#### 3.3.2 数据内容

为启用深度思考的模型回复提供中文字符的推理过程文本（60~200 字），模拟链式推理。

### 3.4 上传区调整（P1）

#### 3.4.1 模型类型决定上传 UI

`Model` 接口新增 `requiresReference: boolean` 字段：

| 模型 | requiresReference | 上传 UI |
|------|------------------|---------|
| DeepSeek V4 Pro | — | 📎 图标按钮（聊天模型通用） |
| GPT-Image-2 | `false` | 📎 图标按钮 |
| Qwen-Image-Max | `true` | **输入框上方直接展示图片上传区** + 文案 "请上传参考图" |
| Gemini 3 Flash | `false` | 📎 图标按钮 |

#### 3.4.2 必填参考图校验

- `requiresReference: true` 的模型，发送前校验是否已上传 ≥1 张参考图
- 未上传时发送按钮 disabled，并显示提示

#### 3.4.3 上传区样式

**必填型：**

- 虚线边框上传区域，位于输入框上方
- 已上传图片显示缩略图 + 移除按钮
- 数量上限 9 张
- 支持 `.jpg,.jpeg,.png,.webp`

**选填型：**

- 操作按钮栏中的 📎 图标按钮
- 点击弹出系统文件选择器
- 上传后在输入框上方显示缩略图小条（可移除）

### 3.5 图片区 / 视频区交互调整（P1）

#### 3.5.1 消耗积分显示

- 图片生成结果卡片顶部 / 底部显示：
  - **消耗积分**：如 `消耗 100 智点`（Mock 数据中每个模型定义不同的消耗值）
  - **生成参数**：比例、质量、优化提示词状态

- 视频生成同理，增加消耗积分、时长、分辨率显示

#### 3.5.2 参考图使用注释

参考图缩略图旁显示使用方式标签：

- `以图生图` — 作为内容参考
- `风格参考` — 作为风格参考
- `角色参考` — 角色一致性参考

用户在参考图列表中可以切换使用方式（当前默认统一为"以图生图"）。

#### 3.5.3 Model 接口扩展

```typescript
interface Model {
  // ...现有字段
  requiresReference: boolean;   // 是否必须上传参考图
  costPoints: number;           // 单次生成消耗智点
  supportedReferences?: ('content' | 'style' | 'character')[];
}
```

### 3.6 移动端适配（P0）

#### 3.6.1 断点定义

- `max-width: 767px`（即 < 768px 为移动端）
- 使用现有 `use-mobile.ts` hook (`MOBILE_BREAKPOINT = 768`)

#### 3.6.2 左侧导航

- 变为固定定位的滑动抽屉（`transform: translateX(-100%)` → `translateX(0)`）
- 通过顶部 hamburger 菜单按钮触发
- 打开时显示半透明遮罩层（点击遮罩关闭）

#### 3.6.3 对话区

- 全宽，内边距缩小（`px-4` 替代 `px-[20%]`）
- 多模型卡片强制 1 列
- 输入框全宽

#### 3.6.4 模型列表

- 横向滚动保留，每行显示更紧凑的卡片（仅图标 + 名称，去掉描述）

## 4. 数据模型变更

interface Message {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  contentType: 'text' | 'markdown' | 'code' | 'image' | 'video' | 'mcp';
  timestamp: Date;
  // V1.2 新增字段

## 7. 版本对比

|--------|------|------|----------|

| **移动端适配** | 基础响应式 | 滑动抽屉、强制单列 | 优化 |

## 1. 概述

V1.2 是一次**架构级升级**，核心变更是从「模型优先」迁移到「对话优先」架构，支持多模型并行对话，同时补齐联网搜索、深度思考、移动端适配等功能。

### 1.1 核心设计原则

1. **对话优先** — 对话是核心实体，模型是对话中可选择的资源
2. **多模型并行** — 单轮对话支持 1~4 个模型同时回复
3. **渐进增强** — 联网搜索、深度思考等功能嵌套在模型卡片内
4. **移动优先响应式** — 768px 断点，左侧导航变为滑动抽屉

| # | 功能模块 | 需求名称 |

## 3. 多模型聚合对话

### 3.1 选模型方式

用户在输入框中输入 `@` 触发模型提及弹窗，可多选 1~4 个模型。已选模型显示为输入框上方 pill 标签。

### 3.2 消息发送流程

1. 用户输入消息 + @模型（1~4 个），点击发送
2. 创建用户消息气泡（右侧，显示 @的模型名称）
3. 为每个被提及的模型创建独立的 AI 回复卡片
4. 模型回复**并行生成**（模拟场景使用 staggered setTimeout：800ms + index × 1200ms）
5. 全部完成后输入框恢复可用

### 3.3 卡片网格布局

```
N=1 → 1 列
N=2 → 2 列
N=3 → 3 列
N=4 → 4 列
```

使用 CSS Grid：`grid-template-columns: repeat(N, 1fr)`

### 3.4 模型回复卡片

```
┌──────────────────────────────────────┐
│ 🤖 DeepSeek V4 Pro    联网 ✓  深度思考 ✓ │
│ ──────────────────────────────────── │
│ [深度思考过程 - 默认折叠]              │
│                                      │
│ 回复正文内容…                          │
│                                      │
│ 引用来源 (3)                          │
│                                      │
│ ⏱ 2.4s    [复制] [重新生成] [回复]     │
└──────────────────────────────────────┘
```

- **回复按钮**：点击后，后续对话只针对此模型（其他模型不再参与）
- **复制 + 重新生成**：仅作用于当前卡片
- **联网/深度思考徽章**：根据本轮发送时的开关状态显示
- **加载态**：显示"思考中…"动画 + 骨架屏

### 3.5 数据结构

```typescript
interface Message {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  contentType: 'text' | 'markdown' | 'code' | 'image' | 'video' | 'mcp';
  timestamp: Date;
  // 新增字段
  modelId?: string;              // assistant 消息关联的模型 ID
  modelIds?: string[];           // user 消息中 @提及的模型 ID 列表
  onlineSearch?: boolean;        // 是否启用联网搜索
  deepThinking?: boolean;        // 是否启用深度思考
  thinkingContent?: string;      // 深度思考过程文本
  searchResults?: SearchResult[];// 联网搜索结果
  responseTime?: number;         // 响应时间（秒）
}

interface SearchResult {
  id: string;
  siteName: string;
  title: string;
  snippet: string;
  url: string;
}
```

## 4. 联网搜索结果展示

### 4.1 UI 表现

- AI 回复卡片底部显示 `引用来源 (N)` 文字按钮（N 为结果总数）
- 点击后**从右侧抽屉**弹出引用来源列表
- 抽屉内容：每条结果包含网站名称、标题、简介、可点击的 URL 链接

### 4.2 抽屉规格

- 宽度：380px
- 标题："引用来源"
- 内容：列表滚动，每条结果带外链图标（↗）

### 4.3 Mock 数据

为 3 个聊天模型的模拟消息各生成 3~5 条 `SearchResult`，包含真实存在的网站名和合理的中文标题/简介。

## 6. 上传区调整

### 6.1 模型类型决定上传 UI

`Model` 接口新增 `requiresReference: boolean` 字段：

| 模型 | requiresReference | 上传 UI |
|------|------------------|---------|
| DeepSeek V4 Pro | — | 📎 图标按钮（聊天模型通用） |
| GPT-Image-2 | `false` | 📎 图标按钮 |
| Qwen-Image-Max | `true` | **输入框上方直接展示图片上传区** + 文案 "请上传参考图" |
| Gemini 3 Flash | `false` | 📎 图标按钮 |

### 6.2 必填参考图校验

- `requiresReference: true` 的模型，发送前校验是否已上传 ≥1 张参考图
- 未上传时发送按钮 disabled，并显示提示

### 6.3 上传区样式（必填型）

- 虚线边框上传区域，位于输入框上方
- 已上传图片显示缩略图 + 移除按钮
- 数量上限 9 张
- 支持 `.jpg,.jpeg,.png,.webp`

### 6.4 上传区样式（选填型）

- 操作按钮栏中的 📎 图标按钮
- 点击弹出系统文件选择器
- 上传后在输入框上方显示缩略图小条（可移除）

## 7. 图片区 / 视频区交互调整

### 7.1 消耗积分显示

- 图片生成结果卡片顶部 / 底部显示：
  - **消耗积分**：如 `消耗 100 智点`（Mock 数据中每个模型定义不同的消耗值）
  - **生成参数**：比例、质量、优化提示词状态

- 视频生成同理，增加消耗积分、时长、分辨率显示

### 7.2 参考图使用注释

参考图缩略图旁显示使用方式标签：
- `以图生图` — 作为内容参考
- `风格参考` — 作为风格参考
- `角色参考` — 角色一致性参考

用户在参考图列表中可以切换使用方式（当前默认统一为"以图生图"）。

### 7.3 Model 接口扩展

```typescript
interface Model {
  // ...现有字段
  requiresReference: boolean;   // 是否必须上传参考图
  costPoints: number;           // 单次生成消耗智点
  supportedReferences?: ('content' | 'style' | 'character')[];
}
```

## 8. 移动端适配

### 8.1 断点

- `max-width: 767px`（即 < 768px 为移动端）
- 使用现有 `use-mobile.ts` hook (`MOBILE_BREAKPOINT = 768`)

### 8.2 左侧导航

- 变为固定定位的滑动抽屉（`transform: translateX(-100%)` → `translateX(0)`）
- 通过顶部 hamburger 菜单按钮触发
- 打开时显示半透明遮罩层（点击遮罩关闭）

### 8.3 对话区

- 全宽，内边距缩小（`px-4` 替代 `px-[20%]`）
- 多模型卡片强制 1 列
- 输入框全宽

### 8.4 模型列表

- 横向滚动保留，每行显示更紧凑的卡片（仅图标 + 名称，去掉描述）

## 10. 实施优先级

| 顺序 | 需求 | 理由 |
|------|------|------|
| **1** | 结构布局调整 + 对话优先迁移 | 基础架构，所有后续需求依赖它 |
| **2** | 多模型聚合对话 | 核心功能，依赖新架构 |
| **3** | 联网搜索 + 深度思考展示 | 依赖多模型卡片结构 |
| **4** | 上传区调整 | 依赖模型字段扩展，可相对独立 |
| **5** | 图片/视频区交互调整 | 依赖模型字段扩展，可相对独立 |
| **6** | 邀请奖励系统 | 独立功能，可并行 |
| **7** | 移动端适配 | 在核心功能完成后统一处理响应式 |

## 11. 不涉及范围

- 不修改现有 MCP 服务中心功能逻辑
- 不修改消费记录 / 支付记录功能
- 不修改账单 / 智点计算逻辑
- 不接入真实 API（全部使用 Mock 数据模拟）
- 不实现真正的用户注册系统（邀请链接为模拟数据）
